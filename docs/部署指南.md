# 星智穹图（AstroVista）部署指南

## 一、环境要求

### 硬件

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 4核 | 8核+ |
| 内存 | 16 GB | 32 GB+（本地模型推理需更多） |
| GPU | 无（纯检索模式可运行） | NVIDIA GPU，≥8 GB 显存 |
| 存储 | 10 GB | 50 GB+（含模型权重和数据集） |

### 软件

| 软件 | 版本 | 安装方式 |
|------|------|---------|
| Python | 3.11+ | https://python.org |
| Node.js | 18+ | https://nodejs.org |
| npm | 9+ | 随 Node.js 安装 |
| Git | 最新版 | https://git-scm.com |
| Docker Desktop | 20+（可选） | https://docker.com |

---

## 二、获取项目

```bash
git clone https://github.com/sherlockhomers/Astro.git
cd Astro
```

---

## 三、选择部署方式

提供两种部署方式，按需选择：

| 方式 | 适用场景 | 操作复杂度 |
|------|---------|-----------|
| **方式A：Docker 一键部署** | 演示、评审、快速体验 | 低（一条命令） |
| **方式B：手动分步部署** | 开发调试、自定义配置 | 中 |

---

## 方式A：Docker 一键部署（推荐用于演示）

### A.1 前置条件

确保已安装 Docker Desktop 并正在运行。

### A.2 配置密钥

在项目根目录创建 `.env` 文件（或直接设环境变量）：

```bash
# Linux/macOS
export AUTH_SECRET=$(openssl rand -base64 48)

# Windows PowerShell
$env:AUTH_SECRET = -join ((48..57)+(65..90)+(97..122) | Get-Random -Count 48 | ForEach-Object {[char]$_})
```

### A.3 一键启动

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

首次构建约需 5-10 分钟（下载基础镜像 + 安装依赖），后续启动约 30 秒。

### A.4 验证启动

```bash
docker ps
```

应看到 6 个容器运行中：

| 容器 | 端口 | 说明 |
|------|------|------|
| `astro-frontend` | 80 | 前端页面入口 |
| `astro-backend` | 8000 | 后端 API |
| `astro-neo4j` | 7474, 7687 | 图数据库 |
| `astro-milvus` | 19530 | 向量数据库 |
| `astro-minio` | 9000, 9001 | 对象存储 |
| `astro-redis` | 6379 | 缓存 |

### A.5 访问

| 服务 | 地址 |
|------|------|
| **前端页面** | http://localhost |
| **后端 API 文档** | http://localhost:8000/docs |
| **Prometheus 指标** | http://localhost:8000/metrics |
| **Neo4j 浏览器** | http://localhost:7474 |
| **MinIO 控制台** | http://localhost:9001（minioadmin/minioadmin） |

### A.6 停止

```bash
docker compose -f docker-compose.prod.yml down
```

## 方式B：手动分步部署（适合开发调试）

### B.1 启动 Docker 基础设施（可选）

```bash
docker compose up -d
```

等待容器状态变为 running（约 1-2 分钟）：

```bash
docker ps
```

如果不需要 Neo4j/Milvus/MinIO/Redis，可跳过此步，系统会自动降级。

### B.2 部署后端

#### 创建虚拟环境

```bash
cd backend
python -m venv .venv
```

#### 激活虚拟环境

```bash
# Windows PowerShell
.venv\Scripts\Activate.ps1

# Windows CMD
.venv\Scripts\activate.bat

# Linux / macOS
source .venv/bin/activate
```

#### 安装依赖

```bash
pip install -r requirements.txt
```

> 国内网络较慢可使用清华镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

#### 配置环境变量

```bash
cp .env.example .env
```

用编辑器打开 `backend/.env`，按需修改：

**必须修改：**

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `AUTH_SECRET` | JWT 密钥，≥32 字符 | dev 模式下留空会自动生成 |

**数据源（决定知识图谱和图像检索是否可用）：**

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `CSV_ROOT` | 天文数据集路径 | `D:/data/天文学数据集.xlsx` |
| `IMAGE_BASE_DIRS` | 图片根目录 | `D:/data/images` |

**Docker 组件启用（执行了 B.1 后设置）：**

| 配置项 | 修改为 |
|--------|--------|
| `NEO4J_ENABLED` | `true` |
| `MILVUS_ENABLED` | `true` |
| `MINIO_ENABLED` | `true` |

**本地大模型（如有权重文件）：**

| 配置项 | 说明 |
|--------|------|
| `STARWHISPER_ENABLED` | `true` |
| `STARWHISPER_MODEL_PATH` | 模型权重路径 |

> 没有模型权重保持 `false`，系统自动降级为纯检索模式，所有页面可正常访问。

**NASA API（Landing 页 APOD 功能需要）：**

| 配置项 | 说明 |
|--------|------|
| `MCP_NASA_API_KEY` | 从 https://api.nasa.gov 免费申请 |

#### 启动后端

```bash
uvicorn app.main:app --reload --port 8000
```

验证：浏览器访问 http://localhost:8000/docs

### B.3 部署前端

打开**新的终端窗口**：

```bash
cd frontend
npm install
cp .env.example .env
```

确认 `frontend/.env` 中：

```env
VITE_API_BASE_URL=http://localhost:8000
```

启动前端：

```bash
npm run dev
```

验证：浏览器访问 http://localhost:5173

### B.4 数据初始化

后端首次启动会**自动**完成以下初始化（需正确配置 `.env`）：

| 步骤 | 条件 | 耗时 |
|------|------|------|
| 知识图谱构建 | `CSV_ROOT` 已配置 | 1-3 分钟 |
| 向量索引构建 | 有文本语料 | 1-2 分钟 |
| Neo4j 同步 | `NEO4J_ENABLED=true` | 30 秒 |
| Milvus 图像索引 | `MILVUS_ENABLED=true` | 2-5 分钟 |

日志出现 `Application startup complete` 表示初始化完毕。

## 四、使用流程

1. 打开前端页面（方式A → http://localhost，方式B → http://localhost:5173）
2. 点击「注册」创建账户 → 「登录」
3. 进入 Dashboard，体验各功能模块：

| 功能 | 入口 | 说明 |
|------|------|------|
| 智能问答 | Dashboard → QA | 输入天文问题，查看流式回答 + Explore Bundle |
| 知识图谱 | Dashboard → Knowledge | ECharts 力导向图探索实体关系 |
| 图像检索 | Dashboard → ImageSearch | 以文搜图 / 以图搜图 |
| 3D 星图 | Dashboard → Starfield | Three.js 行星模型 + 太阳系轨道 |
| Landing 资讯 | 首页 | APOD 每日星图、NeoWs、arXiv 论文 |

## 六、常见问题

### Q1：后端启动报 `torch` 相关错误

安装 CPU 版本：

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
```

### Q2：Docker 容器启动失败

检查端口占用：

```bash
# Windows
netstat -ano | findstr "7474 7687 9000 19530 6379 8000 80"

# Linux/macOS
lsof -i :7474,7687,9000,19530,6379,8000,80
```

### Q3：前端无法连接后端

确认 `frontend/.env` 中 `VITE_API_BASE_URL` 与后端地址一致，且后端已启动。

### Q4：知识图谱为空

确认 `backend/.env` 中 `CSV_ROOT` 指向有效的 Excel 或 CSV 文件。

### Q5：图像检索不可用

需同时满足：Milvus 容器运行 + `MILVUS_ENABLED=true` + `IMAGE_BASE_DIRS` 配置有效图片目录。

### Q6：Landing 页 APOD 显示异常

配置 `MCP_NASA_API_KEY`（从 https://api.nasa.gov 免费申请）。

### Q7：`auth_secret` 警告

dev 环境留空会自动生成随机密钥，无需手动设置。生产环境必须在 `.env` 中配置 ≥32 字符的强密钥。

## 七、停止服务

```bash
# 方式A
docker compose -f docker-compose.prod.yml down

# 方式B
# 前端终端：Ctrl+C
# 后端终端：Ctrl+C
docker compose down  # 停止基础设施容器
```

如需清除所有 Docker 数据（慎用）：

```bash
docker compose -f docker-compose.prod.yml down -v
```
