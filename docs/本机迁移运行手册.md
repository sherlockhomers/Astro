# Astro 本机迁移运行手册

## 1) 已落地的本机路径

- 主知识数据：`F:/天文学数据集/天文学数据集.xlsx`
- 原始图片目录：`F:/astronomical_dataset/astronomical-image-and-csv-dataset/data/images`
- StarWhisper 代码目录：`F:/StarWhisper-main`

后端 `.env` 已按上述路径更新：

- `CSV_ROOT=F:/天文学数据集/天文学数据集.xlsx`
- `IMAGE_BASE_DIRS=F:/astronomical_dataset/astronomical-image-and-csv-dataset/data/images`
- `AUTO_LOAD_IMAGES_CATALOG=true`

## 2) 已完成的迁移动作

- 已重建后端虚拟环境：`D:/Astro/backend/.venv`
- 已安装后端依赖（含 `torch/open-clip/pymilvus/transformers/modelscope`）
- 已启动 Docker 依赖：Neo4j / Milvus / MinIO / Redis
- 已生成图片清单：`F:/天文学数据集/images_catalog.csv`
- 已写入 Milvus 向量（验证样本）：300 条

## 3) 启动命令

后端：

```powershell
cd D:\Astro\backend
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

前端（PowerShell 用 `npm.cmd`）：

```powershell
cd D:\Astro\frontend
npm.cmd run dev
```

## 4) 继续做全量图像向量索引（建议）

当前 Milvus 已有 300 条验证向量，建议继续全量跑完（支持断点续跑）：

```powershell
cd D:\Astro\backend
.\.venv\Scripts\python.exe scripts\index_milvus_clip.py `
  --csv-root "F:/天文学数据集/天文学数据集.xlsx" `
  --batch 32 `
  --start-index 300 `
  --max-images 0 `
  --final-flush-timeout 120
```

## 5) 扩展文本知识库（仅后端，不改前端）

接口方式：

```powershell
curl -X POST http://localhost:8000/api/v1/data/ingest-text `
  -H "Content-Type: application/json" `
  -d "{\"text_root\":\"F:/你的文本目录\",\"category_prefix\":\"text_knowledge\",\"chunk_size\":900,\"overlap\":150}"
```

脚本方式：

```powershell
cd D:\Astro\backend
.\.venv\Scripts\python.exe scripts\ingest_text_corpus.py `
  --text-root "F:/你的文本目录" `
  --csv-root "F:/天文学数据集/天文学数据集.xlsx"
```

## 6) 现阶段说明（重要）

- `F:/StarWhisper-main` 当前是仓库源码目录，不含推理权重分片时，后端会自动回退到检索问答链路。
- 要启用 StarWhisper 模型回答，需要把完整权重目录配置到 `STARWHISPER_MODEL_PATH` / `STARWHISPER_TOKENIZER_PATH`。
