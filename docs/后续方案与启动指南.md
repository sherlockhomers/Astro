# 后续方案与启动指南（保留待完成计划）

本文件用于**保留“待完成/后续怎么做”的计划**，并提供本地启动方式，方便你先看前端页面再逐步把系统做成生产化形态。

## 一、当前阶段已具备的能力（联调演示版）

- **数据接入闭环（前端可操作）**：扫描 CSV → 加载 → 构图 → 查看 schema/path/可视化
- **图像访问规范化**：以 `image_id` 提供图片访问与图库分页
- **检索能力升级（当前为本地向量索引实现）**：支持混合检索结构，为后续 Milvus 接入留好形态
- **GraphRAG trace**：以结构化 trace 输出“意图/检索/子图/路径/引用”
- **轻量指标与压测**：`/api/v1/metrics` 与 `scripts/load_test.py`

## 二、后续待完成计划（你接下来该怎么做）

### M1（图片存储正规化：落 MinIO + 缩略图）

目标：让 75GB/5 万图在 Web 端浏览也不卡。

- **对象存储**：MinIO 存原图 + 多规格缩略图（256/512/1024）
- **分层加载**：前端默认只加载缩略图，点击再取原图
- **离线批处理**：
  - 原图上传 MinIO
  - 生成 `thumb_256/512/1024.webp`
  - 写入图片元数据表（当前用 SQLite，后续可迁移）
- **验收**：
  - 图库分页浏览稳定
  - 首屏缩略图加载快
  - 原图按需加载，不拖垮后端 IO

### M2（真向量检索：Milvus + CLIP/BGE）

目标：以图搜图/以文搜图在 5 万图规模仍能稳定 <1s（视硬件调参）。

- **向量入库**：
  - 图像向量：CLIP/ViT embedding
  - 文本向量：BGE/M3E embedding
  - 写入 Milvus（collection + index）
- **检索链路**：
  - 召回（向量 ANN）→ 融合（RRF/加权）→ 重排序（可选）
  - 支持过滤（类别/来源/时间范围等）
- **验收**：
  - TopK 结果一致性/相关性可解释
  - 增量更新策略明确（新增图片/新增 CSV）

### M3（GraphRAG 真正生产化：LLM + 可点击溯源）

目标：答案有引用、引用可点、trace 可复现。

- **上下文标准化**：子图 + 文本 chunk + 图片样本
- **生成**：LLM（本地或 API）根据上下文生成答案
- **溯源**：
  - citations 对应到实体/图片/关系路径
  - 前端可点击跳转/预览
- **会话**：已做持久化基础（SQLite），后续可迁 Redis/DB

### M4（可观测与压测：找到瓶颈、给指标基线）

- **指标**：p50/p95/p99、吞吐、错误率、MinIO/Milvus/Neo4j 耗时
- **压测**：用 `backend/scripts/load_test.py` 建立基线，逐步优化

## 三、本地启动（先看前端页面）

### 1）启动后端

在 `d:\\Astro\\backend`：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

后端接口文档：`http://localhost:8000/docs`

### 2）启动前端

在 `d:\\Astro\\frontend`：

```bash
npm install
npm run dev
```

前端地址：`http://localhost:5173`

### 3）（可选）启动依赖服务（Neo4j/MinIO/Milvus/Redis）

在 `d:\\Astro`：

```bash
docker compose up -d
```

## 四、快速验收清单（你点开页面就能看）

- 打开 `http://localhost:5173`
- 登录后进入 `/app`
- `知识探索` → `数据接入闭环`：填 CSV 根目录 → 扫描 → 加载 → 构图 → 看 schema/path
- `图像检索` → `图库浏览`：分页查看图片（若已加载 CSV 且含图片字段/路径）
