# 前端功能完整度审计（对照《需求文档》）

本文对照 `需求文档 .md` 的 5.2 模块与 10.1 验收项，评估当前前端实现状态。

## 结论摘要

- 当前前端属于“联调演示版”：核心页面已具备，但数据接入闭环与大规模数据体验仍缺。
- P0 风险在于：缺少可视化的数据导入/构图流程，导致系统能力无法被真实触发。
- P1 风险在于：图像检索未提供分页与虚拟化，75GB/5万图场景无法稳定展示。

## 模块对照（需求 -> 现状）

| 模块 | 需求点 | 现状 | 结论 |
|---|---|---|---|
| 模块一 多模态问答 | F1.1/F1.2/F1.4/F1.5 | `QA.vue` 已对接问答、图像问答、GraphRAG trace | 部分达成（后端为占位逻辑） |
| 模块二 多模态索引 | F2.3/F2.4/F2.5 | `ImageSearch.vue` 提供入口，但无分页/虚拟化 | 页面具备，工程能力不足 |
| 模块三 知识图谱 | F3.1/F3.3/F3.4 | `Knowledge.vue` 有路径/对比/图谱展示；无数据导入流程 | 关键闭环缺失（P0） |
| 模块四 可视化 | F4.1/F4.2/F4.3/F4.5 | 图谱、3D 星图、实体对比已具雏形 | 演示可用，仍需性能治理 |
| 模块五 用户管理 | F5.1/F5.2 | 登录/注册/历史页面存在，路由守卫未启用强鉴权 | 演示可用，生产不可用 |

## 验收项对照（A01-A07）

| 验收项 | 现状 | 风险 |
|---|---|---|
| A01 文本问答 | 前端入口与展示已具备 | 准确率依赖后端占位问答 |
| A02 图像上传识别 | 支持上传并触发接口 | 模型未接入时识别结果弱 |
| A03 多跳推理 | 有路径查询与 GraphRAG trace 展示 | 实际推理链非生产级 |
| A04 图谱可视化 | ECharts 关系图可渲染 | 无导入流程导致真实数据不足 |
| A05 以图搜图 | 页面可操作 | 缺失分页/虚拟化/向量检索 |
| A06 3D星图 | 页面和渲染组件可运行 | 大规模点云未做采样与分层 |
| A07 答案溯源 | 前端可展示 citations/trace | 引用不可点击定位实体/图像 |

## 当前必须优先改造项

1. 在 `Knowledge.vue` 或新页面中补齐数据接入闭环：`scan/load/build/status/schema/paths`。
2. 统一登录态策略：路由守卫 + 退出清理 token + 未登录跳转。
3. 图像检索页面增加分页、加载策略与错误占位，为 5 万图规模做准备。
4. 增加任务状态可视化（导入/构图/索引）以支持工程化运维。
