## 二、项目背景与市场分析

### 2.1 项目背景

随着空间探索技术的飞速发展，天文数据呈爆炸式增长。现有天文知识系统存在以下痛点：

| 痛点 | 现状描述 |
|------|----------|
| 数据孤岛 | 天文图像、元数据、科学文献分散存储，难以关联查询 |
| 检索低效 | 传统关键词搜索无法理解语义，无法处理复杂问题 |
| 可视化单一 | 缺乏交互式、多维度的天体关系可视化 |
| 门槛过高 | 普通用户难以理解专业天文术语和数据 |

### 2.2 竞品分析与差异化

| 维度 | NASA Eyes | Stellarium | WorldWide Telescope | **本项目 AstroGraph** |
|------|-----------|------------|---------------------|------------------------|
| 多模态索引 | ❌ | ❌ | ❌ | ✅ 图像+文本+结构化数据 |
| 知识图谱 | ❌ | ❌ | ❌ | ✅ Neo4j天体知识图谱 |
| 智能问答 | ❌ | ❌ | ❌ | ✅ GraphRAG + LLM |
| 关系推理 | ❌ | ❌ | ❌ | ✅ 图谱路径推理 |
| 中文支持 | 部分 | ✅ | 部分 | ✅ 原生中文 |
| 教育场景 | ✅ | ✅ | ✅ | ✅ 增强交互 |

构建一个**面向天文科普与教育**的多模态知识图谱智能问答系统，实现：

> "让任何人都能通过自然语言或图像，探索浩瀚宇宙的奥秘"

### 5.2 功能模块详细说明

#### 模块一：多模态智能问答 (核心功能)

| 功能点 | 功能描述 | 优先级 | 输入 | 输出 |
|--------|----------|--------|------|------|
| F1.1 文本问答 | 用户输入自然语言问题，系统返回准确答案 | P0 | 文本问题 | 文本答案+来源 |
| F1.2 图像问答 | 用户上传天体图像并提问，系统识别后回答 | P0 | 图像+问题 | 识别结果+答案 |
| F1.3 多轮对话 | 支持上下文理解的连续对话 | P1 | 对话历史 | 连贯回答 |
| F1.4 推理问答 | 支持需要多跳推理的复杂问题 | P0 | 复杂问题 | 推理链+答案 |
| F1.5 答案溯源 | 展示答案来自哪些知识图谱节点/路径 | P1 | 答案 | 溯源图谱 |

**典型问答示例：**

```
┌─────────────────────────────────────────────────────────────────┐
│ 用户问题: "木星最大的卫星是什么？它比月球大多少？"               │
├─────────────────────────────────────────────────────────────────┤
│ 系统处理流程:                                                    │
│ 1. 意图识别: 比较查询 + 属性查询                                 │
│ 2. 图谱检索: (木星)-[拥有卫星]->(卫星集合) 筛选最大              │
│ 3. 属性提取: 木卫三直径5268km, 月球直径3474km                    │
│ 4. LLM生成: 整合信息生成自然语言答案                             │
├─────────────────────────────────────────────────────────────────┤
│ 系统回答:                                                        │
│ 木星最大的卫星是**木卫三(Ganymede)**，直径约5268公里，          │
│ 比我们的月球(3474公里)大约**51.6%**，也是太阳系最大的卫星，     │
│ 甚至比水星还要大！                                               │
│                                                                  │
│ 📊 知识溯源: [木星] --拥有--> [木卫三] --属性--> [直径:5268km]  │
└─────────────────────────────────────────────────────────────────┘
```

### 8.2 GraphRAG 核心架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         GraphRAG 问答流程                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │

│      │                                                                  │
│      ▼                                                                  │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │ Step 1: 问题理解与意图识别                                        │  │
│  │ ├── 实体识别: [木星]                                              │  │
│  │ ├── 意图分类: 关系查询 + 属性查询                                 │  │
│  │ └── 问题改写: 生成图谱查询友好的子问题                            │  │
│  └──────────────────────────────────────────────────────────────────┘  │
│      │                                                                  │
│      ▼                                                                  │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │ Step 2: 多路检索                                                  │  │
│  │ ├── 向量检索: 查找语义相关的文本chunk                             │  │
│  │ ├── 图谱检索: Cypher查询相关实体和路径                            │  │
│  │ │   MATCH (p:Planet {name:'Jupiter'})<-[:EXPLORED]-(s:Spacecraft) │  │
│  │ │   RETURN s.name, s.discoveries                                  │  │
│  │ └── 图像检索: 查找相关图像(可选)                                  │  │
│  └──────────────────────────────────────────────────────────────────┘  │
│      │                                                                  │
│      ▼                                                                  │
│  ┌──────────────────────────────────────────────────────────────────┐  │
│  │ Step 3: 知识融合与排序                                            │  │
│  │ ├── 子图提取: 提取相关的局部知识图谱                              │  │
│  │ ├── 路径推理: 多跳关系推理                                        │  │
│  │ └── 上下文构建: 整合为结构化上下文                                │  │
```
